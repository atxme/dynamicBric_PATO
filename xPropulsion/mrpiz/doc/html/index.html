Programmer pour la cible MRPiZ

Les programmes compilés pour la cible MRPiZ doivent être liés (linked) avec la bibliothèque libmrpiz.a (remarque : la bibliothèque libmrpiz.a dépend de la bibliothèque mathématiques).

Exemple avec gcc : -lmrpiz -lm

Programmer pour le simulateur Intox MRPiZ

Les programmes compilés pour s'exécuter avec le simulateur Intox MRPiZ doivent être liés (linked) avec la bibliothèque libintoxmrpiz.a (remarque : la bibliothèque libintoxmrpiz.a dépend de la bibliothèque libintox.a).

Exemple avec gcc : -lintoxmrpiz -lintox

La compilation de ces programmes doit également définir la macro INTOX pour identifier le mode de compilation pour le simulateur.

Exemple avec gcc : -DINTOX

Pour préciser l'adresse et le port du simulateur Intox, il est possible :

  • soit de préciser les macros suivantes (choix à la compilation) :
    • INTOX_ADDRESS : pour indiquer l'adresse IP du simulateur
    • INTOX_PORT : pour indiquer le port de connexion au simulteur
Exemple avec gcc : -DINTOX_ADDRESS=127.0.0.1 -DINTOX_PORT=12345
  • soit de préciser les variables suivantes dans le code source, avant l'appel à la fonction mrpiz_init() (choix à l'exécution) :
    • char * intox_address : pour indiquer l'adresse IP du simulateur
    • int intox_port : pour indiquer le port de connexion au simulteur
// Exemple en C :
#ifdef INTOX
char * intox_address = "127.0.0.1";
int intox_port = 12345;
#endif
